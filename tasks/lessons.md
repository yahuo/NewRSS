# Lessons

## 2026-03-27

- 当功能需要部署到其他服务器时，不能把核心链路建立在当前机器的个人脚本、skills 目录或本地命令上。先检查运行时依赖是否随仓库一起交付；如果不能，就应当内化到项目里或明确做成可安装依赖。
- 当用户给出外部工具只是作为“参考实现”时，不要把这些工具名、命令名或环境变量直接暴露到 README、接口文案和配置里。先抽象成项目自己的能力，再决定真正需要公开的运行时依赖。

## 2026-04-03

- 当用户已经提供了容器日志和运行时错误时，先以运行时证据为准，再去看本地配置和静态代码。不要因为本地环境里缺少某个变量，就过早把问题归因为“没配 API key”；先确认错误发生在抓取、判定、翻译调用还是落库阶段。
